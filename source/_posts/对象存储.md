## Nodejs中S3 SDK的安装和使用

```bash
npm install @aws-sdk/client-s3 @aws-sdk/s3-request-presigner
```

## 初始化 S3Client

```js
import { PutObjectCommand, S3Client } from "@aws-sdk/client-s3";
import { getSignedUrl } from "@aws-sdk/s3-request-presigner";

const s3 = new S3Client({
    endpoint: "https://oss.wulidisk.top", // OSS 服务地址
    region: "us-east-1", // 任意值，S3 兼容存储不强制校验
    credentials: {
        accessKeyId: "your-access-key", // 访问密钥 ID
        secretAccessKey: "your-secret-key", // 秘密访问密钥
    },
    forcePathStyle: true, // 强制路径样式，对 MinIO 等自建服务必须设为 true
});
```

## 生成预签名上传 URL

```js
const url = await getSignedUrl(
 s3,
 new PutObjectCommand({
  Bucket: "bucket-name",
  Key: "file.js",
  ContentType: "application/javascript",
  Metadata: {
   foo: "bar",
  },
  ChecksumAlgorithm: "SHA256",
  ChecksumSHA256: "content checksum",
 }),
 {
  expiresIn: 6000,
  unhoistableHeaders: new Set(["x-amz-checksum-sha256", "x-amz-meta-foo"]),
 },
);
```

生成URL之后，客户端需要在请求中携带预签名的 URL，请求头中携带预签名的签名信息

## 不同开源OSS服务的区别

> MinIO官方已经停止维护了，推荐使用RustFS，或者使用Minio的2025.4.22这个版本，这是最后一个功能完整的版本

| 特性        | MinIO          | rustfs        |
| ----------- | -------------- | ------------- |
| 编程语言    | Go             | Rust          |
| S3 兼容性   | 完全兼容       | 完全兼容      |
| 许可证      | Apache 2.0     | MIT           |
| 发布版本    | 2015年         | 2020年        |
| 社区活跃度  | 非常活跃       | 活跃          |
| 文档完善度  | 非常完善       | 较完善        |
| 适用场景    | 企业级生产环境 | Rust 项目集成 |
| 性能        | 高性能         | 高性能        |
| 分布式支持  | 支持纠删码模式 | 单节点为主    |
| SDK 支持    | 多语言 SDK     | 主要 Rust SDK |
| Docker 支持 | 官方镜像       | 官方镜像      |
| Kubernetes  | 原生支持       | 社区支持      |

### Minio解决方案

#### Web Console UI

Minio的WebUI被阉割了，现在只是一个简单的文件浏览器，但是开源社区维护了一个超棒的Web Console，下面请出唯一真神：[georgmangold/console](https://github.com/georgmangold/console)

推荐使用Docker启动，使用下面的命令启动：

```yaml
services:
  minio:
    image: minio/minio:RELEASE.2025-09-07T16-13-09Z
    container_name: minio
    restart: always
    command: server /data --console-address :9001 --address :9000
    ports:
      - "9000:9000"
    environment:
      MINIO_ROOT_USER: admin
      MINIO_ROOT_PASSWORD: 12345678
      MINIO_BROWSER: "on"
    volumes:
      - ./data:/data
      - ./certs:/root/.minio/certs
    logging:
      driver: json-file
      options:
        max-file: "10"
        max-size: 5M

  minio-console:
    image: ghcr.io/georgmangold/console
    container_name: minio-console
    restart: always
    ports:
      - "9001:9090"
    environment:
      - CONSOLE_MINIO_SERVER=http://minio:9000
    depends_on:
      - minio
```

使用 `1panel` 安装的可以使用这个脚本

```bash
docker run -d \
  --name minio-console \
  --restart=always \
  --network 1panel-network \
  -p 9001:9090 \
  -e CONSOLE_MINIO_SERVER=http://minio:9000 \
  ghcr.io/georgmangold/console
```
